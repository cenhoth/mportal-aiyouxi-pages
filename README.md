# mportal-aiyouxi-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态资源的形式存放，不依赖特定的后端服务，也不针对任何单个域名或具体网站。仓库的主要目的是提供一种简洁、可复用的页面托管方式，方便在不同场景下进行访问和测试。

## 目录说明

```
mportal-aiyouxi-pages/
├── index.html        # 仓库入口页面（可选）
├── pages/            # 存放主要 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/           # 静态资源（CSS、JavaScript、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md         # 本文件
```

- `pages/`：存放所有独立 HTML 页面文件，每个文件可独立访问。
- `assets/`：存放页面所需的公共资源，按类型归类。
- 根目录下的 `index.html` 可作为导航或欢迎页面，可根据需要添加或修改。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无强依赖关系。每个页面均包含完整的 HTML 结构，可直接在浏览器中打开运行。归档的目的包括但不限于：

- 保存特定功能的页面模板
- 提供测试或演示用的静态页面
- 作为历史版本的快照保存

页面文件命名应尽量简洁清晰，避免使用特殊字符或过长名称。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有需要可同步更新 `index.html` 中的链接。
- 更新页面：直接修改对应 HTML 文件，保留版本历史记录。
- 删除页面：移除文件后，建议同步清理 `index.html` 中的相关链接。
- 资源管理：公共资源（如 CSS、JS 库）建议放在 `assets/` 下，避免在多个页面中重复引用外部链接。

欢迎通过提交 issue 或 pull request 提出改进建议。本仓库以简洁和可维护为优先原则，不追求复杂的功能扩展。
